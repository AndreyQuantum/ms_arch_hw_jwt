# Микросервисы авторизации и публикации сообщений

Данный проект предоставляет два микросервиса, разработанных с использованием FastAPI и PostgreSQL, объединенных с помощью Docker Compose. Проект демонстрирует принципы построения микросервисной архитектуры с использованием JWT-аутентификации.

## Структура проекта

Проект включает в себя два основных микросервиса:

1. **Auth Service (Сервис аутентификации)**
   - Порт: 8000
   - Отвечает за регистрацию пользователей и их аутентификацию.
   - Выдает JWT-токены после успешной проверки учетных данных.
   - Пароли надежно хэшируются с использованием алгоритма bcrypt.

2. **Posts Service (Сервис публикаций)**
   - Порт: 8001
   - Принимает и сохраняет сообщения от имени авторизованных пользователей.
   - Требует обязательной авторизации через валидный JWT-токен, который должен передаваться в заголовке запроса.

## Технологический стек

- Язык программирования: Python 3.12+
- Фреймворк: FastAPI
- ORM: SQLAlchemy
- База данных: PostgreSQL
- Безопасность и хэширование: PyJWT, passlib, bcrypt
- Контейнеризация: Docker, Docker Compose

## Архитектура микросервисов

Каждый микросервис декомпозирован на слои для обеспечения чистоты кода и строгого разделения ответственности:
- `config.py` - загрузка конфигураций и переменных окружения.
- `database.py` - настройка пула подключений к базе данных.
- `models.py` - декларативные модели таблиц SQLAlchemy.
- `schemas.py` - модели Pydantic для строгой валидации входящих запросов.
- `repository.py` - слой для взаимодействия с базой данных (инкапсуляция SQL-запросов).
- `security.py` - логика безопасности (обработка паролей, создание и проверка токенов).
- `main.py` - контроллеры FastAPI, связывающие эндпоинты с логикой.

Таблицы в базе данных инициализируются автоматически при запуске контейнера с помощью SQL-скриптов `init.sql`, расположенных в соответствующих директориях микросервисов.

## Описание API

### Сервис аутентификации (порт 8000)

- `POST /register`
  Регистрация нового пользователя.
  Формат тела запроса (JSON): `{"email": "user@email.com", "password": "securepassword123"}`
  Требование к паролю - не менее 8 символов. При успехе возвращает статус 201 с пустым телом. При дубликате email или простом пароле возвращает статус 400.

- `POST /login`
  Аутентификация пользователя.
  Формат тела запроса (JSON): `{"email": "user@email.com", "password": "securepassword123"}`
  При успехе возвращает статус 200 и JSON объект с токеном: `{"token": "<jwt_string>"}`. При неверных данных возвращает статус 401.

### Сервис публикаций (порт 8001)

- `POST /message`
  Публикация нового сообщения.
  Необходимо передать HTTP-заголовок: `Authorization: Bearer <jwt_string>`
  Формат тела запроса (JSON): `{"message": "Текст сообщения"}`
  При успехе возвращает статус 201 с пустым телом. Возвращает статус 400 при неверной подписи токена, либо 401, если срок действия токена истек.

## Запуск проекта

Убедитесь, что у вас установлены инструменты Docker и Docker Compose. Выполните следующую команду в корневой директории проекта:

```bash
docker compose up -d --build
```

После сборки образов сервисы будут запущены и готовы к обработке запросов. База данных PostgreSQL будет доступна на стандартном порту 5432, а сервисы на портах 8000 и 8001.
